1. 프로젝트 개요목적: 딱딱한 기록 방식에서 벗어나 AI와의 대화를 통해 자연스럽게 하루를 회고하고, 시각적 결과물(이미지)로 기록을 소장하게 함.핵심 가치: 사용자 경험 중심의 기록 자동화, 감정의 시각화, 데이터 자산화.타겟: 일기 작성이 귀찮지만 기록의 필요성을 느끼는 사용자, 자신의 감정을 시각적으로 남기고 싶은 사용자.2. 주요 기능 상세 (User Stories & Requirements)기능 1: 대화형 일기 가이드 (AI 인터뷰어)사용자가
앱을 열면 AI가 먼저 말을 걸어 일기 작성을 유도합니다.초기 페르소나 설정: 사용자의 근황, 기분, 상태를 파악하기 위한 아이스브레이킹 질문 생성.적응형 추가 질의 (Deep Dive):사용자의 답변에서 핵심 키워드(감정, 사건, 인물)를 추출."그때 기분이 어떠셨어요?" 혹은 "그 일이 다음 계획에 어떤 영향을 줄까요?" 같은 구체적 질문으로 심층 정보 확보.완료 로직:충분성 판단: 5W1H 요소가 어느 정도 충족되었을 때 AI가 종료 제안.사용자
명시적 종료: "여기까지 쓸래" 등의 발화 시 즉시 종료.요약 엔진: 전체 대화 로그를 기반으로 3~5문장 내외의 요약문 생성.기능 2: 감정 기반 이미지 생성 (Visualizing)요약된 내용을 바탕으로 오늘 하루를 상징하는 이미지를 만듭니다.프롬프트 엔지니어링: 요약문에서 시각적 묘사(날씨, 표정, 색감, 장소)를 추출하여 Image Gen 모델(DALL-E 3, Stable Diffusion 등)용 프롬프트로 변환.스타일 선택: 사용자가
선호하는 화풍(수채화, 픽셀 아트, 실사 등) 적용 가능.기능 3: 데이터 구조화 및 저장생성된 모든 데이터를 향후 분석이나 회고에 활용할 수 있도록 저장합니다.Raw Data: AI와 나눈 전체 대화 내역 (JSON 포맷 추천).Processed Data: 요약 텍스트 및 감정 태그.Media Data: 생성된 이미지 파일 및 저장 경로.3. 기술 스택 제안 (사용자 숙련도 기반)현재 보유하신 기술셋을 활용한 최적의 조합입니다.구분추천 기술이유BackendSpring
Boot메인 비즈니스 로직 및 사용자 관리, 보안 담당.AI InferencePython (FastAPI)GPU 서버에서 LLM(요약/질의) 및 이미지 생성 모델 서빙.DatabasePostgreSQL + Redis일기 텍스트 저장 및 대화 컨텍스트(Session) 관리.Vector DBQdrant과거 일기 내용을 기반으로 "예전엔 이러셨는데 오늘은 어떠세요?" 같은 개인화 질의 구현 가능.AI gemini
자연어 처리 및 페르소나 유지 능력 탁월.4. 구현 단계별 로드맵 (Step-by-Step)단계 1: 대화 로직 설계 (MVP)단순 질문-답변 구조가 아닌, 이전 대화를 기억하는 Memory 구조 설계.사용자 답변이 일정 길이 미만일 경우 추가 질문을 던지는 루프 구현.단계 2: 요약 및 이미지 연동대화 종료 시 LLM에게 "이 대화를 일기 형식으로 요약하고, 이미지 생성을 위한 묘사를 작성해줘"라고 요청.생성된 묘사를 Python GPU 서버의
모델로 전달하여 이미지 생성.단계 3: 데이터 영속화Spring Boot에서 Python 서버의 결과를 수신하여 DB에 저장.Obsidian 사용 습관을 고려하여, 데이터를 마크다운(.md) 형태로 내보내는 기능 추가 검토.5. 상황별 추천 구현 방식비용 절감을 최우선으로 할 때: 이미지 생성은 로컬 GPU 서버에서 SDXL Turbo 같은 가벼운 모델을 사용하고, LLM은 gemini를 활용하세요.감성적인 품질을 최우선으로 할 때: DALL-E
3를 API로 연동하고, LLM 프롬프트에 "사용자의 감정을 공감해주는 상담가" 페르소나를 강하게 주입하세요.확장성을 고려할 때: 대화 내용을 벡터화하여 저장해두면, 한 달 뒤에 AI가 "저번 주에 고민하시던 일은 잘 해결됐나요?"라고 먼저 물어보는 차별화된 기능을 만들 수 있습니다.
